# Opiflow — Πλάνο προσλήψεων για ασφαλές go-live

> Σκοπός: ποιους προγραμματιστές/ρόλους χρειάζεσαι ώστε να βγεις live **χωρίς προβλήματα** και
> να μην εξαρτάσαι από ένα μόνο άτομο. Είναι γραμμένο με βάση το πραγματικό «στήσιμο» του
> Opiflow (δες `docs/OPIFLOW_OVERVIEW_GR.md`). Για κάθε ρόλο: τι αναλαμβάνει, **γιατί** (ποιο
> πραγματικό ρίσκο καλύπτει), τι να ψάξεις, ερωτήσεις-φίλτρο, και κόκκινες σημαίες.
>
> **Τελευταία ενημέρωση:** 2026-06-25

---

## 0. Η μεγάλη εικόνα πρώτα

Το προϊόν είναι **ήδη φτιαγμένο και σχεδόν έτοιμο** για κυκλοφορία στο web. Άρα **δεν** ψάχνεις
κόσμο για να «χτίσει» — ψάχνεις κόσμο για **αξιοπιστία, συντήρηση και να μη μείνεις εκτεθειμένος**.

Το **#1 ρίσκο** σου σήμερα είναι το **bus-factor**: η βαθιά γνώση του συστήματος (ειδικά της
**τηλεφωνίας** και της υποδομής) δεν είναι μοιρασμένη. Δύο κομμάτια ξεχωρίζουν ως «σπάνια &
επικίνδυνα»:
- Η **τηλεφωνία** (Asterisk/SIP/Twilio/InterTelecom) — σπάνια δεξιότητα, και η ρύθμιση ζει
  **μόνο** στον server, όχι στο git.
- Η **υποδομή/ops** (Vercel/Supabase/Hetzner, μυστικά, backups, χειροκίνητα migrations, καθόλου CI).

Όλα τα υπόλοιπα (web, native, AI, CRM) είναι «κανονικό» full-stack TypeScript — πιο εύκολο να
βρεθεί καλός άνθρωπος.

## 1. Senior Full-Stack TypeScript Engineer  ⭐ (η #1 πρόσληψη — πλήρης απασχόληση)

**Τι αναλαμβάνει:** Γίνεται ο «ιδιοκτήτης του κώδικα». Καταλαβαίνει ολόκληρο το Next.js + React +
Supabase σύστημα, διορθώνει bugs, βγάζει νέα features, κάνει τα deploys, και είναι το άτομο που
ξέρει «πώς δένουν όλα μεταξύ τους».

**Γιατί:** Είναι ο άνθρωπος που σε κάνει **ανεξάρτητο**. Χωρίς αυτόν, κάθε μικρό πρόβλημα μετά
το launch σε μπλοκάρει. Καλύπτει το 80% της καθημερινής δουλειάς.

**Τι να ψάξεις (must-have):**
- Δυνατός σε **TypeScript + Next.js (App Router) + React** (όχι απλώς «έχω δει React»).
- **Postgres / SQL** σε επίπεδο που να γράφει queries και να καταλαβαίνει indexes & migrations.
- **Supabase** (ή τουλάχιστον έχει δουλέψει με Postgres-as-a-service + RLS/auth).
- Έχει **ανεβάσει προϊόν σε production** και έχει κάνει **on-call / διόρθωση live bug**.
- **Ελληνικά** (το προϊόν & οι πελάτες είναι ελληνικά) — βοηθάει πολύ.

**Καλό-να-έχει:** Stripe, webhooks, βασική κατανόηση ασφάλειας (multi-tenant, secrets).

**Ερωτήσεις-φίλτρο (κατάλαβε αν είναι καλός χωρίς να είσαι τεχνικός):**
- «Σε ένα multi-tenant SaaS, πώς εξασφαλίζεις ότι ο πελάτης Α δεν βλέπει δεδομένα του Β;»
  *(Καλή απάντηση: μιλάει για RLS **και** για φίλτρα σε επίπεδο εφαρμογής, ξέρει ότι το
  service-role key παρακάμπτει το RLS.)*
- «Πες μου μια φορά που χάλασε κάτι σε production· πώς το βρήκες και το διόρθωσες;»
- «Πώς θα έβαζες αυτόματο CI σ’ ένα project που σήμερα τρέχει τα tests μόνο τοπικά;»

**Κόκκινες σημαίες:** Θέλει να «ξαναγράψει τα πάντα από την αρχή» πριν καταλάβει το σύστημα·
δεν έχει ακουμπήσει ποτέ production· μπερδεύει frontend-only με full-stack.

## 2. VoIP / Τηλεφωνία Engineer (Asterisk + Twilio + SIP)  📞 (κρίσιμος — contract/part-time αρκεί)

**Τι αναλαμβάνει:** Όλο το τηλεφωνικό κομμάτι: τον **Asterisk** στη Hetzner, το **SIP trunk** με
την **InterTelecom**, τη γέφυρα **Twilio**, την ηχογράφηση, τη δήλωση καταγραφής, και το
«χτύπημα σε κλειστή app». Κυρίως: **βάζει τη ρύθμιση του PBX σε ασφάλεια** (τεκμηρίωση, backup,
ιδανικά infrastructure-as-code) ώστε να μην εξαρτάσαι από ένα μηχάνημα.

**Γιατί:** Είναι το **πιο εξειδικευμένο και πιο επικίνδυνο** κομμάτι. Η ρύθμιση του Asterisk
**δεν είναι στο git** — αν χαλάσει/χαθεί ο server, **σταματάνε όλες οι κλήσεις**. Επίσης
εκκρεμούν πραγματικά τηλεφωνικά θέματα (π.χ. εμφάνιση caller-ID «S» σε εξερχόμενες, προώθηση
κλήσεων από κινητό). *(Σημ.: το «χτύπημα» κλειστής iOS app στις κλήσεις πλέον **δουλεύει**.)*

**Τι να ψάξεις (must-have):**
- **Asterisk / FreePBX** σε βάθος (dialplan, **PJSIP**, MixMonitor recording).
- **SIP trunking** με πάροχο (ιδανικά εμπειρία με **ελληνικό** πάροχο/EETT — σπάνιο αλλά χρυσό).
- **Twilio Programmable Voice** (TwiML, Voice SDK, **VoIP push** / CallKit / PushKit).
- Άνετος σε **Linux server admin** (SSH, systemd, logs, ufw).

**Καλό-να-έχει:** WebRTC/jsSIP, γνώση GDPR για ηχογράφηση κλήσεων στην Ελλάδα.

**Ερωτήσεις-φίλτρο:**
- «Πώς κάνεις ώστε ένα iPhone με **κλειστή** εφαρμογή να χτυπήσει σε εισερχόμενη κλήση;»
  *(Καλή απάντηση: APNs **VoIP** push + PushKit + CallKit — όχι απλό notification.)*
- «Πώς θα έπαιρνες backup και θα ξαναέστηνες έναν Asterisk server από το μηδέν;»
- «Πώς εξασφαλίζεις ότι σε outbound κλήση εμφανίζεται το σωστό caller-ID μέσω SIP trunk;»

**Κόκκινες σημαίες:** Ξέρει «μόνο Twilio» αλλά όχι Asterisk/SIP (ή το αντίστροφο)· δεν έχει
ξαναστήσει trunk με πάροχο· υποτιμά το θέμα του να μην υπάρχει η config σε version control.

> 💡 Αυτός ο ρόλος μπορεί κάλλιστα να είναι **freelancer/σύμβουλος** 1–2 ημέρες/εβδομάδα ή
> retainer — δεν χρειάζεσαι full-time, αλλά χρειάζεσαι **κάποιον δικό σου** πέρα από εσένα.

## 3. DevOps / SRE  🛠️ (σημαντικός — part-time/contract)

**Τι αναλαμβάνει:** Την «υγεία» της παραγωγής: αυτόματο **CI**, **backups** (Supabase + Hetzner),
διαχείριση **μυστικών** (secrets), **monitoring/alerts** (Sentry, uptime), τη **διαδικασία των
migrations**, και ένα σχέδιο **disaster recovery** για το PBX.

**Γιατί:** Σήμερα τα ops είναι χειροκίνητα: **καθόλου CI**, migrations με το χέρι, backups
αδιευκρίνιστα, secrets τα βάζει ο ιδιοκτήτης. Πριν μπει πληρωμένος κόσμος, αυτά πρέπει να
σταθεροποιηθούν — αλλιώς ένα λάθος μπορεί να πέσει «αθόρυβα».

**Τι να ψάξεις:**
- **Vercel + Supabase** εμπειρία· **GitHub Actions** (CI).
- **Linux / Hetzner-style VPS**, backups & monitoring.
- Διαχείριση secrets, βασικό security hardening.

**Ερωτήσεις-φίλτρο:**
- «Έχουμε migrations που τρέχουν με το χέρι στον SQL editor. Πώς θα το έκανες πιο ασφαλές
  χωρίς να σπάσει η παραγωγή;»
- «Τι ακριβώς θα έπαιρνες backup εδώ, και πόσο γρήγορα θα μπορούσες να επαναφέρεις;»

**Κόκκινες σημαίες:** Μόνο «cloud buzzwords» χωρίς να έχει στήσει πραγματικό backup/restore·
θέλει να μεταφέρει τα πάντα σε Kubernetes (overkill για το μέγεθός σου).

> 💡 Συχνά ο **#1 (full-stack)** μπορεί να καλύψει αρκετό από αυτό αν έχει ops διάθεση. Αλλιώς,
> ένας DevOps freelancer για ένα **project 2–4 εβδομάδων** (στήσιμο CI/backups/monitoring) +
> περιστασιακή υποστήριξη είναι ιδανικός.

## 4. Mobile Engineer (React Native / Expo)  📱 (μεσαία προτεραιότητα — contract ή part-time)

**Τι αναλαμβάνει:** Τη native εφαρμογή: **EAS builds & υποβολές** σε App Store/Google Play,
ολοκλήρωση του **iOS non-call push** (ειδοποιήσεις — Firebase-iOS + APNs), και τα native bug fixes.

**Γιατί:** Η native έχει **δικές της παγίδες** (builds μόνο μέσω EAS, δοκιμή σε συσκευή, χωρίς
CI) και **εκκρεμεί** το iOS push **των ειδοποιήσεων** (οι κλήσεις χτυπούν ήδη) & το app icon.
Είναι διακριτή δεξιότητα από το web.

**Τι να ψάξεις:**
- **Expo / React Native** με **EAS builds** & **store submissions** (έχει ανεβάσει app live).
- **iOS push (APNs/PushKit/CallKit)** & **Android FCM** — ιδανικά με εμπειρία σε **VoIP κλήσεις**.

**Ερωτήσεις-φίλτρο:**
- «Έχεις ανεβάσει app σε App Store & Google Play μέσω EAS; Πες μου τη διαδικασία.»
- «Πώς ρυθμίζεις push notifications σε iOS με Firebase;»

**Κόκκινες σημαίες:** Μόνο «Expo Go demos» χωρίς πραγματικό production build/submit· δεν έχει
ασχοληθεί ποτέ με push/credentials.

> 💡 Επικαλύπτεται με τον **#2 (VoIP)** στο κομμάτι του CallKit/VoIP push. Αν βρεις έναν άνθρωπο
> που τα ξέρει και τα δύο, ακόμη καλύτερα.

## 5. (Προαιρετικό/Σύμβουλος) Ασφάλεια & GDPR με ελληνικό πλαίσιο  🔒

**Τι αναλαμβάνει:** Έναν **έλεγχο** πριν το launch: επιβεβαίωση της απομόνωσης δεδομένων
(multi-tenant), νομιμότητα **ηχογράφησης κλήσεων** (δήλωση/συγκατάθεση), υποχρεώσεις **EETT** για
τα νούμερα, και ορθή **διαγραφή δεδομένων** (GDPR).

**Γιατί:** Το προϊόν χειρίζεται **τηλεφωνικές ηχογραφήσεις και προσωπικά δεδομένα πελατών** — σε
ελληνικό/EU πλαίσιο αυτό έχει πραγματικές υποχρεώσεις. Ένας **εφάπαξ** έλεγχος μειώνει νομικό ρίσκο.

**Μορφή:** Σύμβουλος για **εφάπαξ audit**, όχι μόνιμη θέση.

## 8. Πρακτική σύσταση (το ελάχιστο, ρεαλιστικό setup)

Για να βγεις live **σύντομα και ασφαλώς**, χωρίς να υπερ-στελεχωθείς:

1. **1 Senior Full-Stack TypeScript** (πλήρης) — η μόνη «αναγκαία» πλήρης πρόσληψη. Ιδανικά
   κάποιος με ops διάθεση, που να καλύπτει και μέρος του DevOps.
2. **1 VoIP/Asterisk σύμβουλος** (retainer/part-time) — για να ασφαλίσει το PBX και τα τηλεφωνικά
   εκκρεμή. Δικός σου άνθρωπος πέρα από εσένα.
3. **1 DevOps freelancer** για ένα project 2–4 εβδομάδων (CI + backups + monitoring + secrets),
   μετά περιστασιακά.
4. **1 εφάπαξ Security/GDPR audit** πριν τη δημόσια κυκλοφορία.

Το **mobile** μπορεί να περιμένει λίγο (το web MVP βγαίνει πρώτο) ή να καλυφθεί από contractor
για το iOS-push + τα store builds.

> **Ο χρυσός κανόνας:** πρώτα κάλυψε το **bus-factor** στα δύο σπάνια κομμάτια — **τηλεφωνία**
> και **υποδομή/ops**. Εκεί είναι το πραγματικό ρίσκο για ένα «καθαρό» go-live, όχι στα features.

## 9. Onboarding ενός νέου μηχανικού (δώσ’ του αυτά)

Για να μπει γρήγορα **οποιοσδήποτε** νέος, δώσε του με αυτή τη σειρά:
1. `docs/OPIFLOW_OVERVIEW_GR.md` (αυτή η τεχνική επισκόπηση)
2. `PROJECT_STATE.md` (η ζωντανή κατάσταση/changelog)
3. `AGENTS.md` (κανόνες & περιορισμοί)
4. `docs/MVP_READINESS_AUDIT.md` (ο τελευταίος έλεγχος ετοιμότητας)
5. Πρόσβαση: GitHub `gsane3/opiflow`, Vercel, Supabase, Hetzner (PBX), Stripe, Twilio, EAS.

**Πρόταση για «πρώτη δουλειά» που χτίζει εμπιστοσύνη:** στήσιμο **CI** (τα ίδια gates που τρέχουν
σήμερα τοπικά) — μικρό, χρήσιμο, και τον αναγκάζει να καταλάβει το build.
