# PBX-as-code (Asterisk on Hetzner)

> **Why this exists:** today the Asterisk dialplan/PJSIP config lives **only** on the
> box `root@46.224.138.115` and is **not in git** — the #1 single point of failure in
> the whole system (lose the box = lose all call routing). This folder is the
> infrastructure-as-code that makes the PBX **reproducible from zero** so you can
> rebuild it in ~30 minutes instead of reverse-engineering it by hand.
>
> **Scope note (honest):** this is a **scaffold**. The generator (`scripts/provision-asterisk.py`)
> and the carrier setup are already documented (`docs/PBX_SETUP_FOR_INTERTELECOM.md`,
> `docs/CALL_RECORDING_DISCLOSURE.md`). What still needs a one-time **read-only dump**
> from the live box are the hand-applied base config files (trunk registration,
> `from-intertelecom`/`from-twilio`/`from-webrtc` base contexts). Those are marked
> `TODO(extract)` below. Completing them does NOT touch the live box (it's a read-only
> `scp`/`cat`), and is the final step to make this fully turnkey.

## What runs on the box

| Piece | Where | In git? |
|---|---|---|
| Asterisk 20.6 (chan_pjsip) | apt package | playbook installs it |
| Per-business config generator | `/opt/opiflow/provision-asterisk.py` | ✅ `scripts/provision-asterisk.py` |
| Generated per-business includes | `pjsip_opiflow_users.conf`, `extensions_opiflow.conf` | ❌ generated at runtime |
| Base trunk + context config | `/etc/asterisk/*.conf` | ❌ `TODO(extract)` |
| Per-minute provisioning cron | `crontab` (flock-guarded) | ✅ defined in the playbook |
| Secrets (`SIP_CRED_ENC_KEY`, `PBX_WEBHOOK_SECRET`, trunk pw) | `/etc/opiflow/sip.env` | ❌ Ansible Vault (never plaintext) |
| Disclosure audio + ffmpeg transcode | `/var/lib/asterisk/sounds/` | generated by the provisioner |

## Architecture (recap)

```
PSTN ── InterTelecom trunk IT658318 ──> Asterisk (from-intertelecom)
                                           │  Answer → Playback(disclosure) → MixMonitor(record)
                                           ├─> Twilio US1 SIP Domain ─ VoIP push ─> native app
                                           └─> WSS (jsSIP) ─> web softphone
   outbound: app ─ Twilio ─> Asterisk (from-twilio) ─ Gosub(disclosure) ─> InterTelecom (per-DID CLI)
   on hangup: Asterisk ── POST ──> /api/webhooks/voice/pbx (+ /pbx-recording) on Vercel
```

## Rebuild from zero

1. Provision a Hetzner VPS (CX23+ is enough; see `docs/OPIFLOW_OVERVIEW_GR.md` §11 for cost).
2. `ansible-playbook -i ansible/inventory.ini ansible/playbook.yml` (installs Asterisk, deploys
   the provisioner, writes `sip.env` from Vault, installs the per-minute cron).
3. Apply the base config (`TODO(extract)` files) — see `RESTORE_RUNBOOK.md`.
4. Point the InterTelecom trunk + Twilio SIP Domain at the new IP.
5. The per-minute cron regenerates all per-business config from live Supabase within ~1 min.

Full disaster-recovery steps: **`RESTORE_RUNBOOK.md`**.
